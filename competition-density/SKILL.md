# 竞争密度分析 Skill

## 概述

本 skill 完成三件事：
1. **空间计算**（确定性）：对每个目标门店，统计指定半径内的竞争点数量 → 竞争密度得分
2. **LLM 分析**：把得分表交给大模型，生成中文业务分析报告
3. **CSV 导出**：把结果表写成文件，供用户下载

空间计算在工具层完成，LLM 只负责解读数字和生成结论，不参与计算。

---

## 输入规格

| 参数 | 格式 | 说明 |
|------|------|------|
| 目标图层 | CSV | 包含门店名称列 + 经度列 + 纬度列（列名灵活，见下文） |
| 竞争图层 | CSV | 包含经度列 + 纬度列（可含名称列） |
| 评估半径 | 数字 + 单位 | 如 `1km`、`500m`、`1.5公里` |

**列名识别规则**：不强制要求固定列名。读取 CSV 后，按以下优先级识别坐标：

**情况 A：经纬度分列**（优先匹配）
- 经度列：`lng`、`lon`、`longitude`、`经度`（大小写不敏感）
- 纬度列：`lat`、`latitude`、`纬度`

**情况 B：经纬度合并在同一列**（分列识别失败时，自动降级到此）
- 合并列列名候选：`lnglat`、`lonlat`、`coordinates`、`坐标`、`经纬度`
- 若列名未命中，对所有列做值扫描：取第一行值，若匹配 `数字,数字` 模式则认定为合并坐标列
- 拆分规则：逗号前为**经度**，逗号后为**纬度**（`lng,lat` 顺序）

**名称列**：`name`、`store_name`、`门店名`、`名称`、`门店名称`、`id`（找不到时用行号代替）

坐标仍无法识别时，向用户说明并询问列名。

## 执行步骤

### Step 1：读取并验证数据

用 Python 读取两份 CSV，检查：
- 经纬度列是否存在且为数值
- 坐标范围是否合理（经度 −180~180，纬度 −90~90）
- 如有问题，向用户说明并停止

向用户回显："目标门店 N 个，竞争点位 M 个，评估半径 R 公里。"

### Step 2：空间计算（Haversine 公式）

对每个目标门店，遍历所有竞争点，用 Haversine 公式计算球面距离，统计落在半径内的竞争点数量。

**Haversine 公式参考实现**（写入 `scripts/haversine.py` 供复用）：

```python
import math

def haversine_km(lat1, lon1, lat2, lon2):
    R = 6371.0
    phi1, phi2 = math.radians(lat1), math.radians(lat2)
    dphi = math.radians(lat2 - lat1)
    dlambda = math.radians(lon2 - lon1)
    a = math.sin(dphi/2)**2 + math.cos(phi1)*math.cos(phi2)*math.sin(dlambda/2)**2
    return R * 2 * math.asin(math.sqrt(a))

def count_within_radius(target_lat, target_lon, competitors, radius_km):
    return sum(
        1 for _, row in competitors.iterrows()
        if haversine_km(target_lat, target_lon, row['_lat'], row['_lon']) <= radius_km
    )
```

计算结果生成一张表，列为：`门店名称`、`经度`、`纬度`、`竞争点数量`。

### Step 3：计算统计指标

在结果表上追加列：
- `均值偏差`：该门店竞争点数量 − 所有门店竞争点数量的均值（正值 = 高于均值，负值 = 低于均值）
- `竞争强度等级`：分级规则如下——
  - **低**：竞争点数量为 0（无论标准差范围），或数量 < max(均值−1σ, 1)
  - **高**：数量 > 均值+1σ
  - **中**：其余情况
  
  注意：竞争点为 0 时必须归为「低」，不受标准差区间影响。

### Step 4：调用 LLM 生成分析报告

将结果表（含门店名称、竞争点数量、均值偏差、竞争强度等级）作为上下文，调用 LLM，Prompt 结构如下：

```
你是一位零售选址和竞争分析顾问。以下是各门店的竞争密度分析结果：

[插入结果表，文本格式]

全网均值：X 个竞争点，标准差：Y

请用中文完成两项任务：
1. 写一段业务分析（150~250字），说明：
   - 哪些门店竞争最激烈（点名），目前处境如何
   - 哪些门店周边竞争稀少，可能是机会点（点名）
   - 给出一条具体的业务建议
2. 不需要输出表格，表格由系统单独生成。

语气专业但易懂，避免空话。
```

### Step 5：生成竞争热力 Grid

基于竞争图层坐标，生成覆盖所有竞争点的网格，统计每个格子内的竞争点数量，输出 WKT 格式 CSV。

**Grid 参数自动推算**：

```
bbox = (min_lng, min_lat, max_lng, max_lat)  # 竞争图层 bounding box

# 用评估半径作为格子边长（角度近似）
# 纬度方向：1km ≈ 1/111 度
cell_lat = radius_km / 111.0
# 经度方向：1km ≈ 1/(111 * cos(中心纬度)) 度
center_lat = (min_lat + max_lat) / 2
cell_lng = radius_km / (111.0 * cos(radians(center_lat)))

# 在 bbox 四周各外扩半个格子，确保边缘竞争点被完整覆盖
```

**Grid 生成与统计**：

对每个格子：
1. 计算四个角的经纬度坐标
2. 统计落在格子内的竞争点数量（点的经纬度在格子经纬度范围内即算落入，使用简单矩形判断，无需 Haversine）
3. 输出为 WKT POLYGON

**输出 CSV 格式**（两列）：

```
wkt,count
"POLYGON((lng1 lat1, lng2 lat1, lng2 lat2, lng1 lat2, lng1 lat1))",5
"POLYGON((...))",0
...
```

- WKT 坐标顺序：左下 → 右下 → 右上 → 左上 → 左下（闭合）
- 经度在前，纬度在后（GIS 标准顺序）
- count = 0 的格子也保留输出，供渲染时显示空白区域
- 文件名格式：`competition_grid_<YYYYMMDD>.csv`，UTF-8-BOM 编码

### Step 6：输出

**在对话中展示**：
1. 数据摘要（门店数、竞争点数、半径、均值、标准差）
2. LLM 生成的中文分析报告
3. 结果预览表（前 5 行）
4. Grid 摘要：格子总数、格子边长、非零格子数、最大 count 值

**写入文件**（同时提供两个下载）：
- `competition_density_<YYYYMMDD>.csv`：门店竞争密度得分表
- `competition_grid_<YYYYMMDD>.csv`：Grid WKT 热力数据

使用 `present_files` 提供两个下载链接。

## 注意事项

- 半径单位统一转换为公里再计算；支持 `km`、`m`、`公里`、`米`
- 若目标门店数 > 500 或竞争点 > 5000，计算前提示用户可能需要约 N 秒
- LLM 分析只解读数字，不重复罗列全部门店数据
- CSV 输出使用 UTF-8-BOM 编码，确保 Excel 直接打开不乱码
- 不要在对话中输出超过 10 行的完整表格，用"前 5 行预览 + 下载链接"代替
- Grid 格子数上限建议 10000 个；若自动推算超出，向用户说明并适当放大格子尺寸
